# NoemaTrace：让 Agent 执行可回放

Agent 可观测性不应该只停在整体指标上。真正排查问题时，我们需要看到一次执行的路径、上下文、工具调用、失败节点和成本分布。

## 问题从哪里来

Agent 的失败常常不是单点错误，而是一串决策叠加出来的结果。一次重复工具调用可能触发错误级联，一段未使用上下文可能吞掉预算，一个高成本节点可能并没有带来有效信息。

如果没有 trace 回放，这些问题只能靠日志和猜测拼起来。

## NoemaTrace 的切入点

NoemaTrace 选择先从浏览器内离线回放开始。用户拖入 trace JSON，就能查看图谱、时间线、节点详情、token、cost、context budget 和上下文浪费评分。

这种方式的好处是足够轻：不需要后端，不需要账号，也不要求团队先接入复杂平台。它先解决一个具体问题：让一次 Agent 执行可以被看见、被解释、被复盘。

## 可观测性的下一个问题

当单次执行可以回放后，下一步才是沉淀规则：哪些模式表示浪费，哪些工具调用存在风险，哪些失败可以提前预警，哪些执行路径应该进入评测集。

可观测性不是为了展示更多图表，而是为了让 Agent 系统具备持续改进的证据链。
